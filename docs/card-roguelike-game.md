# 千技牌阵小游戏文档

## 概览

这是一个基于 Vite+ 和 React 的前端卡牌闯关小游戏。游戏采用无尽模式和回合制战斗：玩家创建新游戏后获得 10 张起始卡牌，通过出牌攻击怪物并不断闯关。每 10 关出现 Boss。

核心实现位于：

- `src/main.tsx`：React StrictMode 入口、React Router 页面、游戏页、主页、图鉴、设置页、组件渲染。
- `src/game.ts`：卡牌类型、默认卡牌生成、怪物模板、游戏配置、战斗状态和纯游戏逻辑。
- `src/storage.ts`：IndexedDB 存储层、全局卡牌配置、怪物模板、配置快照和最多 8 个存档槽。
- `src/style.css`：战斗界面、卡牌扇面、弹窗、动画、背包布局。

## 卡牌系统

- 系统首次运行时生成 1000 张技能卡，并初始化到浏览器 IndexedDB。
- 后续启动会从 IndexedDB 读取卡池，支持玩家自定义。
- 主页设置页编辑的是全局卡牌配置。
- 新游戏会复制当时的全局卡牌配置，生成独立配置快照。
- 游戏页只读取存档绑定的配置快照，不热替换主页设置。
- 卡牌类型包括攻击、护盾、流血、专注、破防、治疗。
- 卡牌拥有 ID、名称、类型、消耗、强度、稀有度、技能说明和图片 URL。
- 玩家初始获得 10 张卡牌。
- 当前牌组最多 30 张。
- 玩家已获得的所有卡牌会进入卡牌背包集合。

## 战斗规则

- 玩家初始生命为 5。
- 玩家每次从底部扇面手牌中选择卡牌使用。
- 底部战斗手牌最多展示 6 张可用卡牌。
- 每张卡牌在当前牌库循环中只能使用一次。
- 卡牌使用后会从底部手牌中消失。
- 当当前牌组的卡牌全部用完时，玩家扣除 1 点生命，并刷新整组卡牌为可用。
- 生命归零后挑战结束。
- 怪物每次持有 2 张技能卡，按顺序使用；两张用完后重新抽取 2 张。

## 关卡和奖励

- 击败普通怪物后掉落 3 张奖励卡，玩家只能选择 1 张。
- 每 10 关是 Boss。
- 击败 Boss 后同样可以选择奖励卡，或者选择生命上限 +1。
- 如果当前牌组已满 30 张，额外获得的卡牌会转化为生命恢复。

## 背包功能

顶部操作栏提供 `背包` 按钮。背包弹窗分为两部分：

- 当前牌组：显示当前放入牌组的卡牌，支持拖拽排序。
- 背包库存：显示已获得但未放入当前牌组的卡牌。

拖拽规则：

- 从背包库存拖到当前牌组：加入牌组。
- 从当前牌组拖回背包库存：移出牌组。
- 在当前牌组内拖到另一张卡的位置：调整牌组顺序。
- 已使用的卡牌在当前牌组中会标记为 `已用`，仍可拖拽调整或移出。

底部战斗手牌会按照当前牌组顺序，从未使用的卡牌中取前 6 张展示。

## 存档和 Mod 加载

- 主页显示最多 8 个存档槽。
- 创建新游戏时会写入一个 `configSnapshot` 和一个 `saveSlot`。
- 存档保存当前游戏状态，并绑定创建时的配置快照 ID。
- 配置快照包含当时的卡牌、游戏配置和怪物模板。
- 默认存档为配置锁定模式，保证挑战过程不受后续卡牌、怪物或背景配置修改影响。
- 游戏进度会写回 IndexedDB 的存档槽。

## 怪物模板和生成

- 系统首次运行会初始化 10 个怪物模板到 IndexedDB。
- 每个怪物模板包含头部、身体、左手、右手、左腿、右腿 6 个部位。
- 每个部位支持 `SVG/文本` 或 `PNG URL` 两类资源。
- 每个模板可以配置姿态：站立、四肢模式、爬行。
- 每个模板可以配置挂物音频 URL 和怪物背景音乐 URL。
- 进入战斗后，怪物外观从存档快照内的模板随机生成。
- 开启混合模式后，系统会从多个模板随机抽取部位组合成混合怪物。
- 混合怪物默认概率为 10%，可以在游戏配置页调整。

## 游戏配置

- 游戏配置存储在 IndexedDB 的 `gameConfig` 中。
- 支持配置游戏背景、默认音乐 URL、Shader 风格、混合怪物开关和混合概率。
- 默认提供 Ember Grid、Neon Rift、Toxic Wave、Star Forge 四种背景风格。
- 创建新游戏时会把当前游戏配置复制进配置快照。
- 游戏中读取快照内的配置，不会受主页设置的后续改动影响。

## Mod 包

- Mod JSON 可以导出当前卡牌、游戏配置、怪物模板和插件 API 文本。
- 设置页支持导入 Mod JSON。
- 导入时会写入 IndexedDB 中的卡牌、游戏配置和怪物模板。
- 当前插件 API 文本预留为：

```js
export function setup(api) {
  api.registerMonsterTemplate(template);
  api.registerCard(card);
}
```

后续可以在这个基础上扩展 JS 插件执行沙箱和更多注册能力。

## 图鉴和设置

- 主页提供 `图鉴` 和 `设置` 路由入口。

- 图鉴：只读查看所有卡牌，支持按 ID、名称、类型、稀有度和描述搜索。
- 设置：使用子页面管理卡牌、游戏配置、怪物模板和 Mod。
- 卡牌管理：管理 IndexedDB 中的完整卡池，支持搜索、新建、编辑、删除和导出 JSON。
- 设置表单支持配置卡牌的所有属性，包括图片 URL。
- 游戏配置：管理背景、音乐、Shader、混合怪物概率和插件 API 文本。
- 怪物管理：管理 10 个默认怪物模板以及玩家自定义模板。
- Mod：导出或导入包含卡牌、游戏配置和怪物模板的 JSON 包。
- 保存后会写入 IndexedDB，只影响后续新建游戏和主页图鉴。

## 开发者模式

主页提供 `开发者` 入口，进入 `/developer`。

- 开发者模式用于调试怪物、游戏场景、背景音乐、怪物音频和插件内容。
- 页面会读取当前 IndexedDB 中的全局游戏配置和怪物模板。
- 左侧提供游戏场景预览，可以立即查看背景、怪物外观、混合怪物生成效果。
- 音频预览区可以播放游戏背景音乐、怪物背景音乐和挂物音频。
- 场景面板可以修改背景、Shader、游戏音乐 URL、混合怪物开关和概率。
- 怪物调试面板可以切换模板、修改部位资源、姿态和音频 URL。
- 插件内容面板可以编辑插件 API / Mod 入口文本。
- 开发者模式中的修改先保存在页面草稿中，点击 `保存到 index.db` 后才写入全局配置。
- 已经创建的游戏存档继续读取自己的配置快照，不会被开发者模式热替换。

## 界面和交互

- 主页顶部提供 `图鉴`、`设置` 和 `新游戏`。
- 游戏顶部提供 `主页`、`背包`、`记录`、`Help`，不提供设置入口。
- 游戏内顶部标题隐藏，仅保留状态指标和操作按钮。
- `Help` 弹窗收纳规则说明。
- `记录` 弹窗显示完整战斗记录。
- 页面只显示最近的玩家出牌和怪物出牌提示。
- 底部只保留玩家手牌扇面，不显示额外面板标题。

## 动画

- 玩家出牌时，卡牌会从底部扇面弹出并砸向怪物。
- 怪物被命中后会有冲击反馈。
- 奖励卡进入弹窗时有掉落动画。
- 底部手牌使用定位式扇面布局，最多显示 6 张，避免滚动。

## 验证

使用 Vite+ 工作流验证：

```sh
vp check
vp build
vp test
```

当前项目没有测试文件，因此 `vp test` 会以 `No test files found` 退出。
