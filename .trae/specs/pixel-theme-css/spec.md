# Pixel 主题 CSS 规格说明

## 背景

项目需要一个主题扩展系统,在不修改原始 CSS 的情况下提供视觉样式扩展。现有的 style.css 将成为默认主题,同时新的"1-Bit Dither-Punk Horror"主题提供替代美学风格。

## 变更内容

- 将 CSS 自定义属性提取到共享变量文件
- 将公共样式(动画、重置、基础排版)提取到共享层
- 将原始 style.css 重命名为 default.theme.css(保留原始美学)
- 创建 pixel.theme.css 实现 Design.md 中的 1-bit horror 美学
- 为 pixel 主题的卡牌稀有度设计新的 1-bit 美学展示方案

## 影响范围

- 受影响的规格:主题系统扩展
- 受影响的代码:src/style.css → src/themes/default.theme.css, src/themes/pixel.theme.css, src/themes/variables.css

## 新增需求

### 需求:主题变量系统

系统应提供所有可主题化值的 CSS 自定义属性,包括颜色、排版、间距和效果。

#### 场景:变量覆盖

- **当** 主题 CSS 文件定义 CSS 自定义属性时
- **那么** 使用该变量的元素将反映新值

### 需求:Pixel 主题 - 1-Bit Dither-Punk Horror

系统应提供严格单色主题,具有以下特征:

#### 场景:颜色限制

- **当** pixel 主题处于活动状态时
- **那么** 只使用纯黑(#000000)和纯白(#FFFFFF)

#### 场景:抖动背景

- **当** pixel 主题需要深度/渐变效果时
- **那么** 使用 CSS/SVG 抖动图案而非灰度

#### 场景:3层边框

- **当** 在交互元素(按钮、卡片、面板)上渲染边框时
- **那么** 应用 3 层边框:1px 黑色,1px 白色,1px 黑色

#### 场景:按钮反转状态

- **当** 在 pixel 主题中渲染按钮时
- **那么** 正常状态:白色文字在黑色背景上
- **并且** 悬停/激活状态:黑色文字在白色背景上

#### 场景:尖角

- **当** 在 pixel 主题中渲染卡片或容器时
- **那么** 不应用 border-radius(90度尖角)

#### 场景:排版

- **当** 在 pixel 主题中渲染标题时
- **那么** 使用大写文字并增加字间距
- **并且** 使用像素/等宽字体族

#### 场景:"燃烧"发光效果

- **当** 白色元素出现在黑色背景上时
- **那么** 应用微妙的发光效果模拟"燃烧穿透"屏幕

#### 场景:卡顿动画

- **当** 在 pixel 主题中为元素添加动画时
- **那么** 使用步进/滴答式动画实现低帧率复古感

#### 场景:卡牌稀有度标识

- **当** 在 pixel 主题中渲染卡牌稀有度时
- **那么** 使用大块纯色方块配合黑白边框图案来表示不同稀有度
- **并且** 稀有度方块使用抖动填充图案增加视觉层次感
- **具体方案**:
  - 普通(普通): 白色方块带黑色点状抖动
  - 稀有: 黑色方块带白色交叉线图案
  - 史诗: 白色方块带黑色对角线图案
  - 传说: 纯黑方块带白色边框闪烁效果

## 修改需求

### 需求:默认主题保留

原始 style.css 美学应原样保留为 default.theme.css。

## 移除需求

无

## 技术备注

- 主题文件应导入共享变量和重置/样式
- 每个主题文件为 CSS 自定义属性提供覆盖值
- 抖动图案实现为内联 SVG data URI 或 CSS 图案
- 稀有度方块使用 CSS border + background patterns 实现
