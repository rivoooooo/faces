# 主题切换功能规格说明

## 背景

项目已建立主题系统(default.theme.css 和 pixel.theme.css),但目前主题切换需要手动修改 CSS 文件。需要提供在应用设置中切换主题的功能。

## 变更内容

- 在设置界面添加主题切换选项
- 支持切换默认主题(original)和 Pixel 主题
- 主题选择持久化到 localStorage
- 切换主题时动态更新 CSS 导入

## 影响范围

- 受影响的规格:设置界面扩展、主题系统
- 受影响的代码:设置组件、主题加载逻辑

## 新增需求

### 需求:设置界面主题切换

系统应在设置界面提供主题切换选项。

#### 场景:主题选择

- **当** 用户在设置中选择主题时
- **那么** 系统应更新 CSS 导入并持久化选择到 localStorage
- **并且** 页面应立即反映新主题

#### 场景:主题选项

- **当** 用户打开主题设置时
- **那么** 应显示可用主题列表:
  - 默认主题 (original) - 暖色调纸张美学
  - Pixel 主题 (pixel) - 1-bit Dither-Punk Horror 美学

#### 场景:记住选择

- **当** 用户选择主题后刷新页面
- **那么** 系统应从 localStorage 恢复主题选择

## 技术方案

### 方案:动态 CSS 导入

1. 在 HTML 或入口 JS 中添加主题加载逻辑
2. 读取 localStorage 中的主题偏好
3. 动态设置 CSS 链接的 href
4. 设置界面提供单选按钮组切换主题

### 方案:CSS 变量覆盖(备选)

1. 两个主题 CSS 都导入 variables.css
2. 在 HTML 中同时加载两个主题 CSS
3. 使用 CSS 变量覆盖实现主题切换
4. 通过添加/移除特定 class 或覆盖变量实现切换

## 移除需求

无
